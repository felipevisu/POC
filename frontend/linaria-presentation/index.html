Styled Components

What is happening?

The styled-components team announced that the library is now in "maintenance mode": no new features, only critical bug-fixes and security patches.

Why?

Styled-components was built around patterns common in React apps when it was created: runtime CSS-in-JS, heavy use of Context API, runtime style injection etc. But React's direction is shifting: e.g., with Server Components, concurrent rendering, streaming SSR, etc., some assumptions no longer hold

MenuItem.jsx 

import React from 'react';
import styles from './menu.module.css';

export const MenuItem = ({ 
  children, 
  active, 
  disabled,
  onClick 
}) => {
  const className = [
    styles.menuItem,
    active && styles.active,
    disabled && styles.disabled
  ].filter(Boolean).join(' ');

  return (
    <div className={className}>
      {children}
    </div>
  );
};

Dynamic props

Button.jsx 

import React from 'react';
import { styled } from '@linaria/react';

const Button = styled.button`
  font-size: 1rem;
  font-weight: 600;
  border-radius: 4px;
  padding: 0.6rem 1.2rem;
  cursor: pointer;
  border: none;
  transition: background-color 0.2s ease;

  background-color: ${props =>
    props.variant === 'primary'
      ? '#007bff'
      : props.variant === 'secondary'
      ? '#6c757d'
      : props.variant === 'danger'
      ? '#dc3545'
      : '#e0e0e0'};

  color: ${props =>
    props.variant === 'primary' || 
    props.variant === 'danger'
      ? '#fff'
      : '#000'};

  &:hover {
    opacity: 0.9;
  }

  &:disabled {
    opacity: 0.5;
    cursor: not-allowed;
  }
`;

How it works

  1. 1. The babel plugin will look for CSS and styled in the codebase
  2. 2. Then it will generate unique name based on the file hash
  3. 3. Replace the interpolations with CSS custom properties
  4. 4. Generate the CSS file

Original code 

import { styled } from '@linaria/react';
import { families, sizes } from './fonts';

const background = 'yellow';

const Title = styled.h1`
  font-family: ${families.serif};
`;

const Container = styled.div`
  font-size: ${sizes.medium}px;
  background-color: ${background};
  color: ${props => props.color};
  width: ${100 / 3}%;
  border: 1px solid red;

  &:hover {
    border-color: blue;
  }
`;

Parsed code 

import { styled } from '@linaria/react';
import { families, sizes } from './fonts';

const background = 'yellow';

const Title = styled('h1')({

  class: 'Title_t1ugh8t',
  vars: {

Advantages

  1. Selectors are scoped
  2. No pre-processor needed
  3. Automatic removes unused styles
  4. Declarative dynamic stylling with React

vs CSS processors

  1. No new syntax to learn
  2. Same advantages of regular CSS

vs other CSS-in-JS

  1. CSS is downloaded and parsed separately from JS
  2. No extra parse needed
