HTTP — manual → declarative

v19 old hand-rolled


user    = signal<User|null>(null);
loading = signal(false);
error   = signal(false);

constructor() {
  effect(() => {
    const id = this.userId();
    this.loading.set(true);
    this.error.set(false);
    this.http.get<User>(url(id)).subscribe({
      next: u => { this.user.set(u);
                   this.loading.set(false); },
      error: () => { this.error.set(true);
                     this.loading.set(false); }
    });
  });
}

v20 new httpResource


userId = signal(1);

// reactive url → auto refetch
user = httpResource<User>(
  () => `.../users/${this.userId()}`
);

// built-in signals:
//   user.value()
//   user.isLoading()
//   user.error()
//   user.status()

No subscribe, no manual loading/error bookkeeping. URL is a signal computation → re-fetches itself.

v20 opt-in


// app.config.ts
providers: [
  provideZonelessChangeDetection(),
  provideRouter(routes),
  provideHttpClient(),
]

You ask for it. zone.js still shipped.

v21 default


// app.config.ts
providers: [
  provideBrowserGlobalErrorListeners(),
  provideRouter(routes),
]
// no provider needed —
// zone.js gone from deps

Zoneless is the baseline.

Signals notify the scheduler directly. Mutating a plain field from a timer no longer re-renders — that's the whole point of the v20 demo.

Testing — Karma → Vitest

v19 old
Karma + Jasmine. Real browser launcher, slow start, the legacy default.
v20 new
@angular/build:unit-test + Vitest. jsdom, watch mode. Experimental, opt-in.
v21 very-new
Vitest is the default runner. Karma/Jasmine deps dropped entirely.

import { describe, expect, it } from 'vitest';
import { add, isEven } from './calc';

describe('calc', () => {
  it('adds two numbers', () => expect(add(2, 3)).toBe(5));
  it('detects even', () => expect(isEven(4)).toBe(true));
});

DevTools profiling v20+

enableProfiling() emits change-detection & render timings as a custom track in Chrome's Performance panel.


import { enableProfiling } from '@angular/core';

enable() {
  enableProfiling();          // one-way for the session — no public disable
  this.profiling.set(true);
}

// churn ~120 frames: each frame shuffles a signal → CD + re-render,
// so the profiler has real Angular work to record.
churn() {
  const step = () => {
    this.rows.update(r => r.map(n => (n + 1) % 1000));
    if (++this.frames < 120) this.rafId = requestAnimationFrame(step);
  };
  this.rafId = requestAnimationFrame(step);
}

See where CD time goes, framed against your own app code.

Signal Forms v21 · experimental

Forms built on signals — no FormGroup. Model is a plain signal; form() wraps it with a schema of validators.


import { email, form, min, minLength, required, submit }
  from '@angular/forms/signals';

model = signal<SignupModel>({ name: '', email: '', age: null });

f = form(this.model, (path) => {
  required(path.name,  { message: 'Name is required' });
  minLength(path.name, 2);
  required(path.email, { message: 'Email is required' });
  email(path.email);
  min(path.age, 18, { message: 'Must be 18 or older' });
});

// field state is signals:  f().valid()  ·  f.name().errors()  ·  .touched()
async onSubmit() {
  await submit(this.f, async () => { /* server call */ });
}

Signals — the reactive core (v16 → v20)

Everything in this POC sits on these primitives. The shift: from RxJS streams you subscribe to, toward values that recompute themselves.


const count   = signal(0);                 // writable state
const double  = computed(() => count() * 2); // derived, cached, lazy
effect(() => console.log(double()));         // re-runs on dep change

count.set(5);     count.update(n => n + 1);

// linkedSignal: writable, but resets when its source changes (v19)
const selected = linkedSignal(() => options()[0]);

// resource: async state as signals (the engine behind httpResource)
const user = resource({
  params: () => ({ id: userId() }),
  loader: ({ params }) => fetchUser(params.id),   // value / isLoading / error
});

input() · output() · model() replaced the @Input/@Output decorators (v17.1→v19) — component I/O is signals too.

Control flow & @defer (v17+)

Built-in template syntax — no NgIf/NgFor imports. @defer code-splits a block and loads it on a trigger (works client-side, no SSR needed).


@switch (status()) {
  @case ('active') { <p>▶ in progress</p> }
  @case ('done')   { <p>✔ done</p> }
}

@for (c of comments(); track c.id) {
  <li>{{ c.user }}: {{ c.text }}</li>
} @empty { <li>No comments yet.</li> }

@defer (on viewport) {          <!-- own JS chunk, loads on scroll -->
  <app-comments-panel />
} @placeholder { <div>not loaded yet</div> }
  @loading (minimum 300ms) { <div>loading…</div> }
  @error { <div>failed</div> }

The throughline

  • Signals everywhere — state, HTTP, change detection, now forms.
  • Less framework runtime — Zone.js goes from required → optional → gone.
  • Faster feedback — Karma → Vitest; DevTools profiling built in.
  • Declarative over manual — httpResource & Signal Forms replace subscribe + FormGroup boilerplate.

v20 = the pivot point. v21 = the defaults catch up to it.
