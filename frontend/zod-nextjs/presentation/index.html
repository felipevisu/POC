Runtime vs. Compile-time

"TypeScript gives us compile-time safety, but runtime data is the wild west."

What TypeScript thinks we have: 


interface User {
    id: number;
    email: string;
    age: number;
}

What actually comes from the API: 


{
    id: "123",        // ❌ String, not number
    email: null,      // ❌ Null, not string
    birthday: "1990"  // ❌ Wrong field entirely
}

Zod: Your Runtime Type Guardian

Zod is a TypeScript-first schema validation library that bridges the gap between compile-time types and runtime validation.

✅ Zero dependencies
✅ Works in Node.js and browsers
✅ TypeScript inference magic
✅ Composable and chainable
✅ Detailed error messages

"Write your schema once, get TypeScript types AND runtime validation automatically."

From Schema to Safety in One Line

import { z } from 'zod';
// 1. Define your schema
const UserSchema = z.object({
  id: z.number(),
  email: z.string().email(),
  age: z.number().min(18).max(120),
  isActive: z.boolean().default(true)
});

// 2. Get TypeScript type automatically
type User = z.infer;
// Result: { id: number; email: string; age: number; isActive: boolean; }

// 3. Validate runtime data
const result = UserSchema.parse(apiData);
// ✅ Throws if invalid, returns typed data if valid

Notice: We never wrote the TypeScript interface manually!

🔄 Zod Schema with Transformations

const UserSchema = z.object({
    id: z.string().transform(Number),                           // string → number
    name: z.string().transform(s => s.trim().toLowerCase()),    // clean whitespace
    email: z.string().email().transform(s => s.toLowerCase()), // normalize email
    age: z.string().transform(Number),                          // string → number
    salary: z.string().transform(s => 
    parseFloat(s.replace(/[$,]/g, ''))                        // "$75,000.50" → 75000.5
    ),
    tags: z.string().transform(s => 
    s.split(',').map(tag => tag.trim())                       // CSV → string[]
    ),
    isActive: z.string().transform(s => s === 'true'),          // string → boolean
    createdAt: z.string().transform(s => new Date(s))           // string → Date
});

// Parse and transform in one step:
const cleanUser: User = UserSchema.parse(messyApiData);
